# 基于机器学习的美股短期走势预测系统

**课程：** 金融科技
**日期：** 2026 年 3 月

---

## 摘要

本项目构建了一套基于机器学习的美股短期走势预测系统，以 QQQ（纳斯达克 100 ETF）、GOOGL（Alphabet）和 PLTR（Palantir）为标的，预测未来约两周（10 个交易日）的涨跌方向。系统整合了 30 项技术指标与宏观因子特征，采用随机森林、XGBoost 和 LightGBM 三种模型的加权集成策略，并在独立测试集上进行了评估与回测。前端采用 React/Next.js 构建现代化交互界面，后端使用 FastAPI 提供 REST API 服务。实验结果表明，模型在 GOOGL 上取得了 3.42% 的策略收益（同期买入持有为 -7.62%），体现了一定的实用价值；但在 QQQ 和 PLTR 上表现尚有不足，反映出短期股价预测的内在困难。

---

## 一、研究背景与目标

### 1.1 学术意义与实际价值

股票价格预测是金融工程领域的核心问题之一。准确的短期走势预测不仅在学术层面有助于理解市场微观结构和信息扩散机制，在实际层面更直接关系到量化投资策略的盈亏。近年来，随着机器学习技术的成熟和算力成本的下降，越来越多的研究尝试将分类与回归模型应用于金融时间序列预测。

### 1.2 有效市场假说的挑战

尤金·法马（Eugene Fama）于 1970 年提出的有效市场假说（EMH）指出，在一个信息充分流通的市场中，资产价格已经完全反映了所有可获得的信息，因此任何基于历史数据的预测都不应产生超额收益。然而，行为金融学的大量实证研究表明，市场中确实存在动量效应、均值回归、波动率聚集等"异象"（anomaly），这些为机器学习模型提供了潜在的可利用信号。本项目正是在这一学术争论的背景下，探索技术指标和宏观因子组合能否为短期预测提供统计优势。

### 1.3 项目目标

本项目的具体目标是：

1. **数据层面**：从 Yahoo Finance 获取 2021-01-01 至 2026-03-25 期间的日线数据，覆盖目标标的和宏观指标；
2. **特征层面**：构建约 30 个技术指标和宏观因子特征，形成结构化特征矩阵；
3. **模型层面**：训练随机森林、XGBoost、LightGBM 三种模型，并通过验证集 AUC 进行加权集成；
4. **评估层面**：在独立测试集上评估分类性能，并通过模拟交易回测验证策略的实际表现；
5. **应用层面**：构建基于 React/Next.js + FastAPI 的 Web 应用，支持交互式预测和可视化。

## 二、数据来源与预处理

### 2.1 数据源

本项目使用 `yfinance` Python 库从 Yahoo Finance 获取历史行情数据。Yahoo Finance 是全球使用最广泛的免费金融数据源之一，提供经调整的 OHLCV（开盘价、最高价、最低价、收盘价、成交量）日线数据，数据质量可靠且更新及时。

### 2.2 数据范围

- **时间跨度**：2021-01-01 至 2026-03-25，共约 1312 个交易日
- **目标标的**：
  - **QQQ**：Invesco QQQ Trust，追踪纳斯达克 100 指数，代表科技股整体走势
  - **GOOGL**：Alphabet Inc.，全球最大的互联网公司之一
  - **PLTR**：Palantir Technologies，高波动性的数据分析公司
- **宏观/跨资产数据**：
  - **VIX**（^VIX）：CBOE 波动率指数，衡量市场恐慌情绪
  - **TLT**：iShares 20+ Year Treasury Bond ETF，反映长期利率走势
  - **UUP**：Invesco DB US Dollar Index Bullish Fund，反映美元强弱
  - **SPY**：SPDR S&P 500 ETF，用于计算相对强弱

### 2.3 数据完整性检查

在特征工程阶段，系统会自动删除含有缺失值的行（主要来自技术指标的"预热期"，如 50 日均线需要前 50 个交易日来计算）。宏观数据通过前向填充（forward fill）处理非交易日的缺失。

### 2.4 增量更新机制

系统实现了 CSV 文件缓存和增量更新：首次运行时从 Yahoo Finance 下载全量数据并保存为 CSV；后续运行时仅增量获取最新数据并追加至已有文件，大幅缩短数据准备时间。这一设计体现在 `data_fetcher.py` 中的 `fetch_symbol()` 函数：

```python
if path.exists() and not force:
    df = pd.read_csv(path, index_col=0, parse_dates=True)
    last_date = df.index[-1]
    new = yf.download(symbol, start=last_date + pd.Timedelta(days=1), ...)
    if not new.empty:
        df = pd.concat([df, new]).loc[~pd.concat([df, new]).index.duplicated(keep="last")]
```

## 三、特征工程

### 3.1 特征概览

本项目共构建了约 30 个特征，可分为以下四大类别：

| 类别 | 特征数量 | 代表性特征 |
|------|----------|-----------|
| 趋势指标 | ~10 | SMA 比率（5/10/20/50 日）、MACD、MACD 信号线、MACD 柱状图 |
| 动量指标 | ~8 | RSI、随机振荡器（Stoch K/D）、ROC（5/10 日）、Williams %R、日/周收益率 |
| 波动率指标 | ~3 | 布林带百分位（BB_pct）、ATR、20 日波动率 |
| 宏观/跨资产因子 | ~9 | VIX/TLT/UUP/SPY 收盘价及 5 日变化率、QQQ/SPY 相对强弱比 |
| 时间特征 | 2 | 星期几（DayOfWeek）、月份（Month） |

### 3.2 特征设计理由

**趋势指标**：SMA 比率（即收盘价与均线的比值）而非原始均线值，使得特征在不同价格水平上具有可比性。MACD 及其衍生指标捕捉短期与中期动量的交叉信号。

**动量指标**：RSI 和随机振荡器识别超买超卖状态；ROC 直接衡量价格变化速率；Williams %R 提供了另一个维度的超买超卖信号。1 日和 5 日收益率捕捉短期价格动量。

**波动率指标**：布林带百分位标准化了价格相对于波动区间的位置；ATR 衡量绝对波动幅度；20 日滚动标准差提供了波动率的历史视角。

**宏观因子**：VIX 是公认的"恐慌指数"，与股价通常呈负相关；TLT 反映利率环境对股票估值的影响；UUP 捕捉美元走势对跨国企业盈利的压力；SPY 作为大盘基准，用于计算个股相对强弱。

**时间特征**：星期几和月份编码捕捉市场中已知的日历效应（如"周一效应"和"一月效应"）。

### 3.3 目标变量构造

目标变量 `Target` 为二分类标签：

```python
forward_return = df["Close"].pct_change(FORWARD_DAYS).shift(-FORWARD_DAYS)
df["Target"] = (forward_return > 0).astype(int)
```

即：如果从当前日期起未来 10 个交易日的累计收益率大于 0，则标记为 1（看涨），否则为 0（看跌）。

### 3.4 前视偏差的防范

在金融机器学习中，前视偏差（look-ahead bias）是导致模型回测结果虚高的常见陷阱。本项目在以下环节严格防范：

1. **目标变量**：使用 `shift(-FORWARD_DAYS)` 将未来收益率对齐到当前行，并删除最后 `FORWARD_DAYS` 行无法计算目标的数据；
2. **数据切分**：采用严格的时间切分（训练集截止 2025-06-30，验证集截止 2025-12-31，其后为测试集），绝不使用未来数据训练模型；
3. **特征标准化**：`StandardScaler` 仅在训练集上拟合，验证集和测试集使用训练集的统计量进行变换。

## 四、模型选型与训练

### 4.1 模型选型理由

本项目选择了三种基于决策树的集成学习模型：

1. **随机森林（RandomForest）**：Bagging 方法的代表，通过对样本和特征的随机采样降低方差，天然抗过拟合，适合作为稳健的基线模型。

2. **XGBoost**：梯度提升框架的经典实现，通过正则化和列采样有效控制模型复杂度。在 Kaggle 等数据竞赛中表现卓越，尤其擅长处理结构化表格数据。

3. **LightGBM**：微软开发的高效梯度提升框架，采用基于直方图的分裂算法和叶子优先（leaf-wise）生长策略，在保持精度的同时大幅提升训练速度。

选择这三种模型的共同理由是：它们对特征尺度不敏感、能处理非线性关系、支持特征重要性输出，且在金融预测领域有广泛的实证支持。

### 4.2 时间序列交叉验证

金融时间序列具有时序依赖性，不能使用常规的随机 K 折交叉验证。本项目采用 `TimeSeriesSplit`（n_splits=5）进行超参数搜索：

```
Fold 1: [Train: ████] [Val: ██]
Fold 2: [Train: ██████] [Val: ██]
Fold 3: [Train: ████████] [Val: ██]
Fold 4: [Train: ██████████] [Val: ██]
Fold 5: [Train: ████████████] [Val: ██]
```

每一折的训练集始终在验证集之前，确保不会使用"未来信息"。

### 4.3 超参数优化

使用 `RandomizedSearchCV` 进行随机搜索，每个模型搜索 30 次迭代，评估指标为 AUC（ROC 曲线下面积）。超参数搜索空间涵盖了树的数量、深度、学习率、采样比例和正则化强度等关键参数。例如 XGBoost 的搜索空间：

| 超参数 | 候选值 |
|--------|--------|
| n_estimators | 100, 200, 300, 500 |
| max_depth | 3, 5, 7, 9 |
| learning_rate | 0.01, 0.05, 0.1, 0.2 |
| subsample | 0.6, 0.7, 0.8, 0.9 |
| colsample_bytree | 0.5, 0.6, 0.7, 0.8 |
| reg_alpha | 0, 0.1, 1 |
| reg_lambda | 1, 2, 5 |

### 4.4 集成学习策略

三个模型训练完成后，以各模型在验证集上的 AUC 为权重进行软投票（soft voting）集成：

```python
weights = {name: auc / total_auc for name, auc in val_aucs.items()}
ensemble_proba = sum(w * model.predict_proba(X)[:, 1] for model, w in ...)
```

这种加权集成策略赋予验证集上表现更好的模型更大的话语权，通常能获得比单一模型更稳定的预测效果。

## 五、实验结果与分析

### 5.1 模型性能对比

以下为各标的在独立测试集（2026 年 1 月至 3 月）上的分类性能：

**QQQ 测试集性能：**

| 模型 | Accuracy | AUC | F1 |
|------|----------|-----|-----|
| RandomForest | 0.702 | 0.464 | 0.000 |
| XGBoost | 0.561 | 0.522 | 0.359 |
| LightGBM | 0.632 | 0.466 | 0.000 |
| Ensemble | 0.544 | 0.472 | 0.235 |

**GOOGL 测试集性能：**

| 模型 | Accuracy | AUC | F1 |
|------|----------|-----|-----|
| RandomForest | 0.474 | 0.497 | 0.423 |
| XGBoost | 0.544 | 0.409 | 0.278 |
| LightGBM | 0.667 | 0.683 | 0.457 |
| Ensemble | 0.579 | 0.471 | 0.333 |

**PLTR 测试集性能：**

| 模型 | Accuracy | AUC | F1 |
|------|----------|-----|-----|
| RandomForest | 0.368 | 0.585 | 0.526 |
| XGBoost | 0.351 | 0.391 | 0.519 |
| LightGBM | 0.351 | 0.542 | 0.507 |
| Ensemble | 0.368 | 0.496 | 0.526 |

### 5.2 回测结果

回测基于以下交易策略：集成模型概率 > 0.6 时全仓买入，0.5~0.6 时半仓，<= 0.5 时空仓（现金）。每 10 个交易日调仓一次，交易成本设为 0.1%。

| 标的 | 策略收益 | 买入持有收益 | 夏普比率 |
|------|---------|-------------|---------|
| QQQ | -1.11% | -4.01% | -1.75 |
| GOOGL | +3.42% | -7.62% | 1.82 |
| PLTR | -10.20% | -7.68% | -0.77 |

### 5.3 未来两周预测

基于截至 2026-03-25 的最新数据，集成模型给出以下预测：

| 标的 | 方向 | 集成看涨概率 |
|------|------|-------------|
| QQQ | 看跌 | 29.0% |
| GOOGL | 看涨 | 52.4% |
| PLTR | 看涨 | 68.0% |

### 5.4 结果分析

**GOOGL 为何表现最好？** GOOGL 在测试期间的 LightGBM 单模型表现突出（AUC = 0.683），且回测策略收益 +3.42% 显著优于买入持有的 -7.62%。可能原因有：（1）作为大型蓝筹科技股，GOOGL 的价格走势更加"规则"，技术指标的信号可靠性更高；（2）LightGBM 捕捉到了 GOOGL 在 2026 年初的阶段性下跌趋势，成功执行了"减仓回避下跌"的策略。

**PLTR 为何回测差但预测信号强？** PLTR 作为高波动性成长股，价格走势受市场情绪和突发事件驱动较多，技术指标的预测能力有限。测试期内模型 Accuracy 仅约 0.35~0.37，说明模型经常误判。但当前 68.0% 的看涨概率反映的是最新一期数据的特征状态——PLTR 可能正处于某种技术面超卖或动量回升的信号区域。需要注意的是，高概率并不意味着高置信度，模型在该标的上的历史准确率本身就不高。

**模型在下跌市场中的表现特点。** 测试期（2026 年初）恰逢市场调整——QQQ 买入持有亏损 4.01%，GOOGL 亏损 7.62%。在这种环境下，模型的核心价值不在于精准预测方向，而在于通过"不确定时持有现金"的策略规避部分下跌风险。QQQ 策略亏损 1.11%（优于买入持有的 -4.01%）和 GOOGL 策略盈利 3.42% 都体现了这一"防守型"优势。但 PLTR 的案例（策略 -10.20% vs 买入持有 -7.68%）也说明，当模型信号频繁出错时，主动交易反而会因交易成本和错误时机而产生额外损失。

## 六、软件工程实践

### 6.1 系统架构

项目采用前后端分离架构：

- **前端**：Next.js 14 + TypeScript + Tailwind CSS，搭配 TradingView Lightweight Charts（专业 K 线图）、Recharts（统计图表）和 Framer Motion（动画过渡）
- **后端**：FastAPI（Python）提供 REST API，负责数据获取、模型推理和回测计算
- **ML Pipeline**：data_fetcher、features、model、backtest 等核心模块保持不变，由 FastAPI 调用

视觉设计采用黑色 + 红橙色（#E8522A）主题配色，标题使用京华老宋体字体，整体风格简洁专业。

### 6.2 启动方式

项目使用 `uv` 管理 Python 依赖，前端使用 `pnpm` 管理 Node.js 依赖。用户只需在项目根目录运行 `./start.sh`，即可一键启动前后端服务：启动时系统会自动刷新最新行情数据，随后在浏览器中打开完整的预测和可视化界面。

### 6.3 项目结构设计


### 6.4 代码模块化

系统采用清晰的模块化设计，各模块职责单一：

- **config.py**：所有硬编码参数集中管理，修改标的、时间范围或模型参数只需改一处；
- **data_fetcher.py**：负责数据获取和持久化，上层模块无需关心数据来源细节；
- **features.py**：纯粹的特征工程逻辑，输入原始价格数据，输出特征矩阵和标签；
- **model.py**：封装了从训练到预测的完整流程，支持单独训练和批量训练；
- **backtest.py**：独立的回测引擎，可脱离模型训练流程单独运行。

### 6.5 缓存策略

系统在多个层面实现了缓存以优化性能：

1. **数据层**：CSV 文件缓存 + 增量更新，避免重复下载；启动时自动刷新最新数据；
2. **模型层**：训练好的模型通过 `joblib` 序列化为 `.pkl` 文件，FastAPI 启动时直接加载至内存。

### 6.6 新增功能

相比初始版本，系统新增了以下交互功能：

1. **预测周期选择器**：用户可在界面上选择 5 天、10 天或 20 天的预测周期，系统根据所选周期动态调整模型推理和回测参数；
2. **启动时自动数据刷新**：每次启动应用时，系统自动从 Yahoo Finance 获取最新行情数据，确保预测基于最新市场状态；
3. **专业 K 线图表**：使用 TradingView Lightweight Charts 渲染交互式 K 线图，支持缩放和十字光标；
4. **动画过渡**：使用 Framer Motion 实现页面元素的流畅动画效果。

## 七、局限性与未来改进

### 7.1 当前局限

1. **数据量有限**：约 1312 个交易日的数据对于机器学习模型来说偏少，尤其是测试集仅覆盖约 2-3 个月，统计显著性不足。

2. **信息来源单一**：仅使用了价格和成交量衍生的技术指标以及少数宏观因子。缺少基本面数据（财报、估值指标）、新闻情感分析和社交媒体情绪等可能包含增量信息的数据源。

3. **对突发事件无能为力**：机器学习模型的预测依赖于历史模式的重复。对于突发的地缘政治事件、央行政策转向、公司丑闻等"黑天鹅"，模型没有任何应对能力。

4. **概率校准不足**：模型输出的概率值未经校准，52.4% 的看涨概率并不严格意味着"52.4% 的概率上涨"，而只是模型的一个决策分数。

5. **交易策略过于简单**：回测中使用的固定阈值（0.5/0.6）和固定调仓周期（10 天）未经优化，更精细的仓位管理和风控策略可能显著改善实际表现。

### 7.2 改进方向

1. **更多数据源**：接入新闻 API（如 NewsAPI）进行情感分析，利用 NLP 模型（如 FinBERT）提取新闻情绪特征；加入财报发布日期和关键财务指标。

2. **概率校准**：使用 Platt Scaling 或 Isotonic Regression 对模型输出概率进行校准，使其更接近真实的后验概率。

3. **多时间框架**：同时构建日线、周线、月线级别的特征，让模型在不同的时间尺度上捕捉信号。

4. **深度学习探索**：尝试 LSTM、Transformer 等序列模型，直接建模时间序列的时序依赖关系。

5. **更严格的回测框架**：引入滑点模型、更真实的交易成本结构和资金管理规则，使回测结果更贴近实盘。

---

## 八、结论

本项目完成了一个从数据获取、特征工程、模型训练到交互式可视化的完整美股预测系统。在技术实现上，项目展示了如何用模块化的 Python 代码组织机器学习 pipeline，如何通过时间序列交叉验证和严格的数据切分避免前视偏差，以及如何通过 React/Next.js + FastAPI 的前后端分离架构将预测模型封装为用户友好的现代化 Web 应用。

在预测效果上，结果是诚实而复杂的。GOOGL 上的策略表现令人鼓舞（在下跌市场中实现正收益），说明技术指标特征配合集成模型在某些标的上确实能捕捉到有价值的信号。但 QQQ 和 PLTR 上的表现不佳也提醒我们：股票市场的短期走势受到极其复杂的因素驱动，仅凭历史技术指标很难在所有标的和所有市场环境中稳定盈利。

总体而言，本项目在"学以致用"的维度上达到了预期目标——它不是一个可以直接用于实盘交易的系统，而是一个完整展示机器学习在金融预测中应用流程的教学案例。它同时也展示了一个重要的教训：在金融领域，承认模型的局限性与追求模型的准确性同样重要。
