# ORACLE Stock Predictor 用户使用指南

本指南面向完全没有技术背景的金融科技学生，帮助你快速上手这个基于机器学习的美股预测工具。

---

## 一、环境准备

在启动应用之前，请确保你的电脑已安装以下工具：

| 依赖 | 用途 | 安装方式 |
|------|------|----------|
| **uv** | Python 包管理器，用于运行后端 (FastAPI) | `curl -LsSf https://astral.sh/uv/install.sh \| sh` |
| **Node.js / npm** | JavaScript 运行时，用于运行前端 (Next.js) | 从 https://nodejs.org 下载安装 LTS 版本 |

首次使用前，还需在 `frontend/` 目录下运行一次 `npm install` 来安装前端依赖。

---

## 二、如何启动和访问

本项目采用 **React/Next.js 前端 + FastAPI 后端** 的前后端分离架构。前端负责界面展示和交互，后端负责数据获取、模型推理和回测计算。

### Mac / Linux（终端 Terminal）

只需一行命令即可同时启动前端和后端：

```bash
cd /path/to/stock-predictor
./start.sh
```

启动成功后，终端会显示：

```
  ORACLE 已启动
  前端: http://localhost:3000
  后端: http://localhost:8000
  Ctrl+C 停止
```

按 **Ctrl+C** 即可同时停止前端和后端。

### Windows（命令提示符 / PowerShell）

Windows 下需要打开**两个终端窗口**，分别启动后端和前端：

**终端 1 -- 启动后端：**

```bash
cd C:\path\to\stock-predictor
uv run python backend.py
```

**终端 2 -- 启动前端：**

```bash
cd C:\path\to\stock-predictor\frontend
npm run dev
```

关闭时，在两个终端中分别按 Ctrl+C。

### 浏览器访问

打开浏览器，访问 **http://localhost:3000** 即可看到应用界面。

> 注意：后端会自动获取最新的市场数据，无需手动更新数据文件。

## 三、界面全面介绍

整个界面采用黑色背景搭配明亮的红橙色 (#E8522A) 作为主色调，使用京华老宋体 (KingHwa OldSong) 字体，整体风格沉稳有力。界面分为左侧的 **侧边栏**（控制面板）和右侧的 **主区域**（图表与结果展示）。

### 侧边栏

侧边栏是你操作这个工具的控制中心，从上到下依次有以下功能：

#### 1. ORACLE 标识

侧边栏顶部展示 ORACLE 项目标识。

#### 2. 选择标的（Symbol Selector）

三个按钮，分别对应三个可选标的：

| 代码 | 全称 | 说明 |
|------|------|------|
| **QQQ** | Invesco QQQ Trust | 追踪纳斯达克100指数的ETF，涵盖苹果、微软、英伟达等100家科技巨头。买QQQ相当于"一键买入美国科技股大盘"。 |
| **GOOGL** | Alphabet Inc. (Google) | Google母公司的股票，全球最大的搜索引擎和广告平台公司。 |
| **PLTR** | Palantir Technologies | 一家专注大数据分析和人工智能的公司，以政府和企业级数据平台闻名，波动性较大。 |

**为什么选这三个？** QQQ代表大盘指数，波动适中；GOOGL是大型蓝筹科技股，相对稳定；PLTR是高波动成长股。三者覆盖了不同风险等级，能测试模型在不同市场环境下的表现。

点击按钮即可切换标的，主区域的行情数据和K线图会随之更新。

#### 3. 时间窗口滑块（Time Window）

一个可拖拽的滑块，范围从 **30天** 到 **500天**。

- **数值小（30-60天）：** 只显示最近一两个月的走势，适合观察短期波动和近期趋势。
- **数值中等（120-200天）：** 半年左右的走势，能看到中期趋势的形成和转折。
- **数值大（300-500天）：** 一年以上的走势，适合观察长期趋势、大的支撑和阻力位。

拖动滑块后，主区域的K线图会实时更新显示范围。

#### 4. 预测周期选择（Prediction Horizon）

三个按钮，可选择预测的时间范围：

| 选项 | 含义 |
|------|------|
| **5天** | 预测未来一周的涨跌方向 |
| **10天** | 预测未来两周的涨跌方向 |
| **20天** | 预测未来一个月的涨跌方向 |

不同的预测周期会影响模型的训练目标和预测结果。短周期的预测受短期噪声影响更大，长周期的预测则更偏向中期趋势判断。

#### 5. 生成预测按钮

点击红橙色的 **"生成预测"** 按钮后，系统会依次完成以下操作：

1. 读取该标的最新的历史价格数据
2. 计算技术指标（均线、RSI、MACD等）作为特征
3. 将特征输入已训练好的模型（RandomForest、XGBoost、LightGBM）
4. 输出该标的未来涨跌方向及概率
5. 自动运行回测，展示模型历史表现
6. 展示特征重要性分析

整个过程大约需要几秒到十几秒。

### 主区域

#### 1. 行情概览（Hero Section）

主区域顶部展示当前所选标的的核心行情信息：

- **标的名称：** 当前选中的股票或ETF的全称
- **最新价格：** 当前最新的收盘价
- **日涨跌幅：** 相较前一个交易日的涨跌百分比。上涨时显示为橙色，下跌时显示为灰色
- **52周范围（52W Range）：** 过去一年内的最低价和最高价区间，帮助你了解当前价格处于历史什么位置

#### 2. K线图（蜡烛图）

K线图采用 TradingView 图表组件，分为上下两部分。

**上半部分：蜡烛图 + 均线**

每根"蜡烛"代表一天的价格变动：

- **橙色蜡烛（上涨）：** 收盘价高于开盘价，即当天股价上涨。蜡烛实体的底部是开盘价，顶部是收盘价。
- **灰色蜡烛（下跌）：** 收盘价低于开盘价，即当天股价下跌。蜡烛实体的顶部是开盘价，底部是收盘价。
- **上下细线（影线）：** 代表当天的最高价和最低价。

图上还有两条均线：

- **SMA 20（20日简单移动平均线）：** 过去20个交易日的平均收盘价。它反映短期趋势。当股价在SMA 20上方运行，说明短期走势偏强。
- **SMA 50（50日简单移动平均线）：** 过去50个交易日的平均收盘价。它反映中期趋势。
- **金叉与死叉：** 当SMA 20从下方穿过SMA 50向上，称为"金叉"，通常被视为看涨信号；反之从上方穿过向下，称为"死叉"，被视为看跌信号。

**下半部分：成交量柱状图**

每根柱子代表当天的成交量（交易了多少股）。颜色与蜡烛一致 -- 橙色代表当天上涨，灰色代表当天下跌。

- 成交量突然放大，说明市场参与者增多，可能有重要事件发生。
- 价格上涨伴随放量，说明上涨有资金支持；价格上涨但缩量，可能后劲不足。

**鼠标交互操作**

- **悬停：** 将鼠标移到任意一根蜡烛上，会显示该天的开盘价、收盘价、最高价、最低价。
- **缩放：** 滚动鼠标滚轮可以放大/缩小图表。
- **拖拽：** 按住鼠标左键拖拽可以平移图表查看不同时间段。

#### 3. 预测结果区域

点击 "生成预测" 按钮后，K线图下方会出现预测结果，包含以下内容：

**预测卡片**

一个醒目的卡片，展示模型集成后的核心判断：

- **方向（Direction）：** 显示为"看涨"或"看跌"。看涨时以橙色显示，看跌时以灰色显示。
- **概率（Probability）：** 集成模型给出的上涨概率百分比。例如"68%"意味着模型综合判断后，认为上涨的概率是68%，下跌的概率是32%。概率越接近50%，说明模型越"犹豫"，信号越弱；越接近100%或0%，信号越强。

**模型信号（Model Signals）**

展示三个底层模型各自的预测结果，以水平概率条的形式呈现：

| 模型 | 说明 |
|------|------|
| **RF (RandomForest)** | 随机森林。用100棵决策树各自独立判断后投票，不容易过拟合，预测稳健。 |
| **XGB (XGBoost)** | 梯度提升树。每棵新树专门纠正前面的错误，一轮一轮迭代改进，通常表现很强。 |
| **LGBM (LightGBM)** | 轻量级梯度提升树。与XGBoost思路相同但算法更高效，训练速度更快。 |

每个模型各自输出一个概率条，你可以对比不同模型的判断：

- 如果所有模型都指向同一方向，信号更可信。
- 如果模型之间分歧很大（比如有的60%看涨，有的45%），说明市场信号不明确。

**雷达图与指标表格**

- **雷达图：** 以图形化方式展示模型在多个维度上的表现，直观呈现模型的综合能力。
- **指标表格：** 列出各模型的详细评估指标，包括：

| 指标 | 含义 |
|------|------|
| **Accuracy（准确率）** | 模型预测正确的比例。55%意味着100次预测中有55次方向判断正确。 |
| **AUC（曲线下面积）** | 衡量模型区分"涨"和"跌"能力的综合指标，范围0到1。0.5相当于随机猜，越接近1越好。 |
| **F1 分数** | 精确率和召回率的调和平均值，综合考虑查准率和查全率。 |
| **Precision（精确率）** | 当模型说"看涨"时，实际真的涨了的比例。 |
| **Recall（召回率）** | 所有实际上涨的日子里，模型成功预测到了多少。 |

**这些数字多高算好？**

在金融预测领域，不要期望看到90%以上的准确率。股市受无数因素影响，能达到以下水平已经相当不错：
- **准确率 53-58%：** 在金融领域已经有实际价值
- **AUC 0.55-0.65：** 说明模型有一定的区分能力
- **任何指标超过60%：** 在真实市场中已经是非常好的信号

#### 4. 回测结果区域

**什么是回测？**

回测就是"用历史数据模拟交易"。假设你在过去某段时间内，完全按照模型的预测来买卖股票，最终的收益会是多少？回测帮你在不花真金白银的情况下，检验策略在历史上是否有效。

**四个指标卡片：**

界面以四张独立卡片的形式展示核心回测数据：

- **策略收益（Strategy Return）：** 按照模型信号交易的总收益率。比如显示"+25.30%"，意味着如果按模型操作，资金增长了25.3%。
- **买入持有收益（Buy & Hold Return）：** 如果你什么都不做，一开始就买入并持有到最后的收益率。这是策略的"对照组" -- 你的策略至少要跑赢买入持有才有意义。
- **最大回撤（Max Drawdown）：** 在回测期间，从最高点到最低点的最大跌幅。比如"-15.20%"意味着期间你的账户曾经从最高点下跌了15.2%。最大回撤衡量的是你可能承受的最大亏损，越小越好。
- **夏普比率（Sharpe Ratio）：** 衡量"每承担一单位风险能获得多少超额收益"。参考标准：
  - < 0：策略在亏钱
  - 0 - 1.0：一般
  - 1.0 - 2.0：不错
  - \> 2.0：很好

**收益曲线图（Equity Curve）：**

一张折线图，对比策略表现和买入持有表现。如果策略曲线长期在买入持有曲线上方，说明ML策略跑赢了大盘。

#### 5. 特征重要性

一张水平柱状图，展示模型在做预测时最看重哪些数据特征。柱子越长，该特征对预测结果的影响越大。

**这个图在告诉你什么？**

它揭示了模型"认为"影响股价涨跌最关键的因素是什么。你可以据此理解模型的决策逻辑 -- 它不是一个黑盒子，你能看到它在依据什么做判断。

**常见特征名含义对照：**

| 特征名 | 中文含义 | 简单解释 |
|--------|----------|----------|
| RSI_14 | 14日相对强弱指标 | 衡量近期涨跌的强弱程度，超过70可能超买，低于30可能超卖 |
| MACD | 移动平均收敛发散 | 通过两条不同周期均线的差值判断趋势方向和动量 |
| MACD_Signal | MACD信号线 | MACD的平滑版本，与MACD交叉时产生买卖信号 |
| MACD_Hist | MACD柱状图 | MACD与信号线的差值，正值看涨、负值看跌 |
| SMA_20 | 20日均线 | 过去20天的平均收盘价 |
| SMA_50 | 50日均线 | 过去50天的平均收盘价 |
| EMA_12 | 12日指数移动平均 | 比SMA更侧重近期数据的均线 |
| EMA_26 | 26日指数移动平均 | 26天的指数均线 |
| BB_Upper / BB_Lower | 布林带上轨/下轨 | 价格波动的上下边界，突破可能意味着异常波动 |
| BB_Width | 布林带宽度 | 衡量市场波动率，宽度收窄可能预示大行情即将到来 |
| ATR_14 | 14日平均真实波幅 | 衡量价格波动剧烈程度 |
| VIX_Close | VIX收盘价 | "恐慌指数"，反映市场对未来波动的预期 |
| VIX_ROC_5 | VIX 5日变化率 | VIX最近5天的变化幅度，急升通常意味着市场恐慌加剧 |
| Volume_ROC | 成交量变化率 | 成交量相比之前的变化百分比 |
| Close_ROC_5 | 5日价格变化率 | 最近5天的涨跌幅 |
| Close_ROC_10 | 10日价格变化率 | 最近10天的涨跌幅 |
| Day_of_Week | 星期几 | 一周中的第几天（周一到周五），某些日子历史上可能有规律 |

## 四、如何解读预测结果做决策

以下是一些实用的参考建议（仅供学习，不构成投资建议）：

### 信号强度参考

| 集成概率 | 信号强度 | 建议 |
|----------|----------|------|
| > 65% 看涨 | 较强看涨 | 可以考虑轻仓做多，但务必设止损 |
| 60-65% 看涨 | 中等看涨 | 可以小仓位尝试，结合其他分析判断 |
| 55-60% 看涨 | 弱看涨 | 信号偏弱，谨慎操作 |
| 50-55% | 模糊地带 | 信号太弱，不建议操作，观望为主 |
| 55-60% 看跌 | 弱看跌 | 信号偏弱，不建议轻易做空 |
| > 60% 看跌 | 较强看跌 | 可以考虑减仓或观望 |

### 综合判断方法

1. **看模型一致性：** 如果三个模型（RF、XGB、LGBM）的概率条都指向看涨，比只有一个看涨更可信。
2. **看回测表现：** 如果该标的的回测策略收益长期跑赢买入持有，说明模型对这个标的的预测能力更强。
3. **看特征重要性：** 如果当前市场环境中，排名靠前的特征正在发出极端信号（比如RSI超过80），要额外注意。

### 小资金策略建议

如果你是学生或刚开始投资，资金有限：

- 永远不要把全部资金投入单一预测。
- 建议单次投入不超过总资金的10-20%。
- 先用模拟盘验证一段时间，再考虑真实操作。
- 每笔交易设好止损位（比如亏损5%就卖出）。

### 风险提醒

- 模型基于历史数据训练，但市场会受到突发事件影响（财报、政策、地缘政治等），这些无法被历史数据预测。
- 回测收益不代表未来收益。过去表现好不意味着未来一定好。
- 不要在单一信号上重仓操作。

## 五、常见问题

### 为什么模型准确率只有 50-55%？这正常吗？

完全正常。股市是一个极其复杂的系统，受海量信息影响，本质上具有很强的随机性。学术界普遍认为，在股市预测中，能稳定达到53-58%的准确率已经非常有价值。即使只有55%的准确率，只要配合合理的仓位管理和风控策略，长期下来依然可以盈利。不要拿90%以上的准确率作为期望值 -- 那几乎不可能在真实市场中实现，如果一个模型声称能做到，大概率是过拟合了。

### 为什么预测"看涨"但股票实际跌了？

这是完全正常的情况。即使模型给出68%的看涨概率，也意味着有32%的概率是跌的。概率预测不是确定性预测 -- 它告诉你的是"更可能"发生什么，而不是"一定会"发生什么。就像天气预报说"70%概率下雨"，你带了伞但最终没下雨，不能说天气预报是错的。正确的做法是关注长期胜率，而不是纠结某一次的对错。

### 数据多久更新一次？

后端（FastAPI）在每次启动时会自动通过 Yahoo Finance 获取最新的市场数据，无需手动下载或更新CSV文件。只要重新启动应用（或重新运行 `./start.sh`），数据就会刷新到最新状态。

### 可以预测其他股票吗？

当前版本只支持 QQQ、GOOGL、PLTR 三个标的。如果你想预测其他股票，需要：

1. 在后端代码中添加该股票的数据获取逻辑
2. 用 `src/` 中的训练脚本为该股票训练新模型
3. 将训练好的模型保存到 `models/` 目录
4. 在前端和后端代码中添加新的股票代码选项

这需要一定的编程基础。未来版本可能会支持自定义添加标的。

### 启动报错怎么办？

- **"uv: command not found"：** 需要先安装 uv，参考第一节的环境准备。
- **"npm: command not found"：** 需要先安装 Node.js，参考第一节的环境准备。
- **端口被占用：** 如果 3000 或 8000 端口已被其他程序占用，需要先关闭占用端口的程序，或者修改相应的配置。
- **前端依赖未安装：** 首次运行前，请先在 `frontend/` 目录下执行 `npm install`。
